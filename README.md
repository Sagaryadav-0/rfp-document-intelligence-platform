# Compliance Project

This repository contains a React + Supabase application for processing RFP PDFs and generating structured Excel workbooks.

Security notes:
- The password gate is validated by the Supabase edge function using `APP_PASSWORD`.
- The password itself must be configured in environment variables, not hardcoded into the frontend.
- This project now uses browser-persisted session IDs for data isolation.
- Uploaded PDFs are stored in a private Supabase Storage bucket under `/session_id/filename.pdf`.
- Excel downloads use signed URLs (1-hour expiry) for secure access.
- Row-level security (RLS) is enabled on the `rfp_sessions` table and storage bucket, restricting access by `session_id`.

Admin Dashboard:
- Access at `/admin` using the admin password.
- Requires `ADMIN_PASSWORD` environment variable.
- Shows all RFP sessions with file details, session IDs, and timestamps.

To set up:
1. Run the migration: `supabase db push`
2. Set environment variables: `APP_PASSWORD`, `SUPABASE_SERVICE_ROLE_KEY`, `ADMIN_PASSWORD`
3. Deploy functions: `supabase functions deploy`

### Run the CLI

```bash
supabase bootstrap
```

Or using npx:

```bash
npx supabase bootstrap
```

The bootstrap command will guide you through the process of setting up a Supabase project using one of the [starter](https://github.com/supabase-community/supabase-samples/blob/main/samples.json) templates.

## Breaking changes

We follow semantic versioning for changes that directly impact CLI commands, flags, and configurations.

However, due to dependencies on other service images, we cannot guarantee that schema migrations, seed.sql, and generated types will always work for the same CLI major version. If you need such guarantees, we encourage you to pin a specific version of CLI in package.json.
